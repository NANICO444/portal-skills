1 O que é esta pasta

Esta pasta é o centro nervoso do seu setup OpenCode. Ela reúne 9 fontes de conhecimento absorvidas de projetos como Kiro, Vercel Labs, Ultra Prompt Nexus, oh-my-openagent, Aurora, Hefesto, Taste Skill e Impeccable — tudo sintetizado, organizado e pronto para uso.

Em vez de ter 9 projetos separados que você nunca lembra onde estão, tudo converge aqui: skills no formato OpenCode, agentes especializados, regras de comportamento e 3 sistemas completos para cenários diferentes.

2 Para que serve

🎯 Dar super-poderes ao OpenCode

Cada skill é uma instrução especializada que o OpenCode carrega automaticamente. Em vez de você ter que pedir "faça code review com segurança OWASP" e explicar o que é — a skill já traz a metodologia pronta. É como ter dezenas de manuais de referência dentro do seu terminal.

🧠 Decidir mais rápido e melhor

Com o sistema de decisão, você pode classificar qualquer pergunta e receber uma resposta em 30-60 segundos com justificativa e ação: deve investir? qual stack usar? é arriscado? como organizar o processo? Os agentes decisores usam modelos leves (MiniMax-M3, DeepSeek Flash) para responder sem custo.

🔍 Supervisionar código com rigor

O sistema max-thinking tem 12 agentes de auditoria que examinam código em 9 dimensões (segurança, performance, cobertura de testes, dependências, padrões, docs, linter, fix suggester, code review extremo). A skill supervisor devolve um veredito estruturado: aprovado/rejeitado, score, o que apagar, o que adicionar, o que fazer, o que baixar.

🤖 Orquestrar agentes headless

O sistema multi-agent roda 2 servidores OpenCode headless (portas 3001 e 3002), cada um com seus sub-agentes especialistas. Um orquestrador delega tarefas: code review, testes, debug, refatoração, segurança, otimização, pesquisa, documentação, arquitetura e DevOps.

4 Como funciona na prática

O OpenCode já está configurado para carregar automaticamente skills, agentes e regras desta pasta através de junction points no diretório global: 

~/.config/opencode/skills/  →  junction para 181 skills
~/.config/opencode/agents/ →  junction para agentes max-thinking
~/.config/opencode/rules/  →  regras always-* aplicadas em todo chat

É automático. Você só precisa reiniciar o OpenCode. Em qualquer chat novo, ele já vem com tudo carregado. Skills são ativadas por palavra-chave no seu prompt — você não precisa nem lembrar o nome delas.

⚡ Dica: Se quiser rodar o OpenCode apontando para um sistema específico: 
opencode --dir "skills pastas melhorias agentes\multi-agent"
opencode --dir "skills pastas melhorias agentes\decision-system"
opencode --dir "skills pastas melhorias agentes\max-thinking-system"

8 Notas importantes

📍 As pastas-fonte (.kiro/, .agents/, ULTRA_PROMPT_ANTIGRAVITY/, SKILLS/, oh-my-openagent/, etc.) não devem ser modificadas. Elas são a origem do conhecimento — as skills sintetizadas estão em multi-agent/.opencode/skills/.

📍 181+ skills não é exagero. Cada skill é um arquivo .md com frontmatter YAML que o OpenCode interpreta automaticamente. Você pode ver a lista em INDEX.md.

📍 Junction points são links simbólicos do sistema. Se o OpenCode parar de reconhecer skills ou agentes, verifique se os junctions em ~/.config/opencode/ ainda apontam para esta pasta.

📍 Os 3 sistemas compartilham skills, mas têm configurações independentes (LSP, MCP, regras). Se você modificar um agente no multi-agent, ele não afeta o max-thinking.

# Onboarding Checklist — Novo Dev

## Dia 1: Setup

- [ ] Conta de email criada
- [ ] Acesso ao GitHub/GitLab
- [ ] Acesso ao Slack/Discord da equipe
- [ ] Documentacao de onboarding lida
- [ ] Variaveis de ambiente configuradas

## Semana 1: Conhecer o Codigo

- [ ] Repo clonado
- [ ] Dependencias instaladas
- [ ] Testes rodando local
- [ ] Primeiro "good first issue" atribuido
- [ ] Pair programming com senior (4+ horas)
- [ ] PR de typo/docs aberto (PR de boas-vindas)

## Semana 2: Primeiro PR de Codigo

- [ ] Branch criada do padrao do projeto
- [ ] Convensoes de codigo seguidas
- [ ] Tests incluidos
- [ ] CI passou local
- [ ] Code review recebido
- [ ] PR mergeado

## Mes 1: Conhecer a Arquitetura

- [ ] Diagramas de arquitetura revisados
- [ ] Servicos principais mapeados
- [ ] Banco de dados conhecido
- [ ] Deploy process conhecido
- [ ] On-call rotation explicado

## Recursos

- [ ] Wiki / Notion / Docs URL salvo
- [ ] Style guide da equipe
- [ ] Coding conventions
- [ ] Definition of Done
- [ ] Padroes de comunicacao (PRs, issues, Slack)

## Expectativas

- 30 dias: productive, PRs simples
- 60 dias: features completas
- 90 dias: on-call rotation
- 6 meses: mentor de novos

## Feedback

- 1 semana: como estao sendo os primeiros dias?
- 1 mes: ha bloqueios? O que esta confuso?
- 3 meses: nivel de conforto? Onde precisa de ajuda?

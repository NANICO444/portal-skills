# Mapeamento Das Personas De Aurora

O Kiro IDE usa instrucoes permanentes, skills e subagentes. Por isso, as personas do Aurora original foram traduzidas para esses formatos.

| Persona Original | Forma No Pacote | Quando Atua |
|---|---|---|
| UI Designer, sempre ativa | `AGENTS.md` e `00-aurora-identidade.md` | Em qualquer tarefa de interface |
| UX Architect, sempre ativa | `AGENTS.md` e `02-processo-de-design.md` | Ao organizar fluxo e estrutura |
| Brand Guardian | `aurora-brand-guardian` | Marca, identidade e consistencia |
| Content Creator | `aurora-content-creator` | Texto de interface e landing page |
| SEO Specialist | `aurora-seo-content` | Pagina publica e SEO basico |
| Visual Storyteller | `aurora-visual-storytelling` | Narrativa de pagina ou produto |
| Paid Media | `aurora-paid-media` | Criativos e paginas de campanha |
| Accessibility Auditor | `aurora-accessibility-audit` | Gate de UI antes de entrega |
| Performance Benchmarker | `aurora-performance-audit` | Desempenho percebido e tecnico |
| Reality Checker | `aurora-reality-check` | Revisao critica final |

## Mudancas Necessarias

- Nenhuma persona delega para outro agente.
- Backend necessario pode ser feito pela propria Aurora por meio de `aurora-fullstack-feature`.
- Revisoes bloqueadoras devem ser informadas diretamente ao usuario.
- Publicacao, credenciais, gastos e acoes irreversiveis continuam exigindo confirmacao.

# Biblioteca De Componentes

## Ordem Recomendada

1. Primitivos: texto, botao, link, campo, icone e container.
2. Estados: hover, focus, disabled, loading, erro e vazio.
3. Padroes compostos: formulario, card, modal, navegacao e tabelas.
4. Telas: componha usando primitives existentes.

## Criterios

- API simples e coerente com o framework existente.
- Acessibilidade por padrao: labels, foco, semantica e teclado.
- Variacoes justificadas por uso real, nao por antecipacao.
- Tests ou exemplos para comportamento interativo relevante.

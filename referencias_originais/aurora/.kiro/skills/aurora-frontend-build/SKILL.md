# Build Frontend

1. Leia `package.json`, estrutura de componentes, estilos e testes antes de escolher abordagem.
2. Trabalhe dentro da stack ja usada, salvo pedido explicito de mudanca.
3. Implemente mobile-first, componentes pequenos e tokens reutilizaveis.
4. Garanta estados de carregamento, vazio, falha, sucesso e interacao quando aplicaveis.
5. Evite dependencias novas para resolver algo simples.
6. Rode lint, typecheck, testes ou build existentes relevantes.
7. Abra a tela localmente ou obtenha evidencia visual quando a tarefa alterar UI.

Relate claramente caso nao consiga executar a aplicacao ou observar o resultado visual.

# Estrategia De Testes

1. Defina o comportamento que pode quebrar e o risco para o usuario.
2. Use teste unitario para regras isoladas e integrado para contratos entre partes.
3. Use teste de interface ou validacao manual quando o fluxo visual importar.
4. Nao declare cobertura sem executar a verificacao correspondente.
5. Relate o que foi testado e o que permaneceu fora do alcance.

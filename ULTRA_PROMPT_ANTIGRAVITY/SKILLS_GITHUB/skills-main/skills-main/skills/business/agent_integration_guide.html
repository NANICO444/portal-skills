Como conectar a biblioteca de 56K tokens ao agente de forma eficiente — gastando apenas 6-15K tokens por sessão.

O Problema

Por que simplesmente "jogar os arquivos" é caro

⚠️
Se o agente carregar todos os 11 arquivos de uma vez: ~56.000 tokens de contexto em cada mensagem. Para GPT-4o ou Claude Sonnet, isso pode custar R$2-8 por conversa. Em 100 sessões de mentoria: R$200-800 desnecessários.

A Solução

Sistema de 3 camadas: Roteador → Skill → Script

🗺️

Camada 1 — AGENTS.md

915 tokens. Sempre carregado. Contém o mapa completo: qual arquivo para qual situação. O agente lê isso e para. Só vai adiante se precisar de detalhes.

📄

Camada 2 — Skill Files

2.300–8.200 tokens cada. Carregados sob demanda. Para 95% das perguntas, 1-2 arquivos são suficientes. Nunca mais que 3 por sessão.

⚡

Camada 3 — Scripts Python

~50 tokens de output. Para CNAE: executar o script em vez de ler o código-fonte (7.445 tokens). 150× mais eficiente.

✅
Resultado: Custo típico por sessão de mentoria = 915 (roteador) + ~9.000 (2 skills) = ~10.000 tokens. Economia de 82% vs carregar tudo.

Integração 1

Claude Code (claude.ai/code ou CLI)

Claude Code lê o arquivo CLAUDE.md na raiz do projeto automaticamente a cada sessão. A estratégia é criar um CLAUDE.md mínimo que aponta para a biblioteca.

1

Clonar o repositório de skills

Clonar uma vez, fica disponível localmente para todas as sessões.

2

Criar CLAUDE.md na raiz do seu projeto

Claude Code carrega isso automaticamente — sem custo extra de instrução.

3

Pronto — o agente sabe onde buscar

Quando você faz uma pergunta tributária ou empresarial, ele lê o skill correto e responde com profundidade.

# No terminal do seu projeto:

git clone https://github.com/Joehott/skills.git ./skills

# Criar CLAUDE.md na raiz do projeto:
cat > CLAUDE.md <<'EOF'
# Skills de Negócio e Tributação — Brasil

Você tem acesso a uma biblioteca de skills especializadas em:
- Análise e planejamento de negócios
- Tributação brasileira completa (MEI → SA Capital Aberto)
- Reforma Tributária 2026-2033

## REGRA: Carregar apenas o necessário
Antes de responder qualquer pergunta de negócio ou tributação:
1. Leia: `skills/business/AGENTS.md` (915 tokens — sempre primeiro)
2. O AGENTS.md dirá qual arquivo carregar para a tarefa específica
3. Nunca carregue mais de 3 arquivos de skill por sessão

## Para CNAE:
Execute o script — não leia o código-fonte:
`python3 skills/business/brazil_cnae_database.py <codigo>`
`python3 skills/business/brazil_cnae_database.py --search "<termo>"`

## Para descobrir qual skill carregar:
`python3 skills/business/skills_loader.py "<descreva a tarefa>"`
EOF
💡
Se preferir não clonar, adicione no CLAUDE.md o caminho via GitHub raw: https://raw.githubusercontent.com/Joehott/skills/main/skills/business/AGENTS.md — Claude Code pode fazer WebFetch para buscar sob demanda.

Integração 2

OpenAI Codex / OpenCode

Codex e OpenCode leem o arquivo AGENTS.md na raiz do repositório. O arquivo já foi criado na biblioteca com o nome correto.

# Opção A — Clonar e usar localmente:
git clone https://github.com/Joehott/skills.git ./skills

# O AGENTS.md já está em skills/business/AGENTS.md
# Para que o Codex encontre na raiz do projeto:

# Opção B — Apenas o AGENTS.md como system prompt:
# Copie o conteúdo de AGENTS.md e cole no system prompt do agente.
# 915 tokens fixos no contexto — barato e sempre presente.
⚠️
No Codex, se o agente não tiver acesso ao sistema de arquivos local, use os URLs raw do GitHub. O AGENTS.md já contém os links — o agente pode fazer fetch do arquivo necessário quando precisar.

Integração 3

Hermes / ChatGPT / Gemini / qualquer chat

Para agentes sem acesso ao sistema de arquivos, a abordagem é injetar o contexto relevante manualmente antes da pergunta.

1

Modo Rápido — Cole 1 skill no início da conversa

Identifique o assunto, vá ao GitHub, copie o conteúdo do arquivo relevante e cole antes da sua pergunta. Custo: 1 arquivo = ~4-8K tokens.

2

Modo Eficiente — Cole só o AGENTS.md

Cole o AGENTS.md (915 tokens) no início da conversa e diga ao agente para buscar o arquivo correto via WebFetch quando precisar.

3

Modo Sistema — AGENTS.md no System Prompt

Se a plataforma permite System Prompt customizado, cole o AGENTS.md lá. Ele estará disponível em todas as conversas sem custo adicional por mensagem.

Ferramenta

skills_loader.py — O roteador inteligente

Agentes com acesso ao terminal podem usar o roteador para descobrir automaticamente qual skill carregar, sem depender de lógica interna.

# Descobrir quais skills carregar para uma tarefa:
python3 skills/business/skills_loader.py "abrir empresa tech no simples nacional"

# Output (exemplo):
Skills recomendadas (2):
  1. brazil_tax_regulatory.md — 3967 tokens
  2. brazil_tax_complete.md   — 8243 tokens
  Economia: 43.790 tokens vs carregar tudo

# Carregar uma skill específica direto no stdout (para pipe):
python3 skills/business/skills_loader.py --load brazil_tax_regulatory

# Listar todas as skills disponíveis:
python3 skills/business/skills_loader.py --list

# Baixar um arquivo do GitHub se não tiver local:
python3 skills/business/skills_loader.py --fetch brazil_tax_reform_2026_2032

Armadilha comum: o agente lê o arquivo brazil_cnae_database.py inteiro (7.445 tokens) achando que vai encontrar a resposta. Errado. Ele deve executar o script e ler o output (50 tokens).

Fluxo Ideal

Como o agente deve se comportar em uma sessão

# 1. Usuário faz uma pergunta de negócio ou tributação
User: "Qual regime tributário é melhor para minha empresa de SaaS faturando R$800K/ano?"

# 2. Agente lê o roteador PRIMEIRO (915 tokens, rápido)
Read: skills/business/AGENTS.md
→ Roteador diz: para "regime tributario" → carregar brazil_tax_regulatory.md

# 3. Agente carrega APENAS a skill necessária
Read: skills/business/brazil_tax_regulatory.md  (3.967 tokens)

# 4. Se precisar calcular alíquota por faturamento:
python3 skills/business/brazil_cnae_database.py --simular --cnae 6311900 --faturamento 800000
→ Output: alíquota efetiva 11.54%, DAS R$92.320/ano, comparativo Presumido R$114.640 (50 tokens)

# 5. Responde com precisão usando dados reais da skill

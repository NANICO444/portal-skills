# Testes Para Entrega

1. Descubra framework e comandos de teste existentes.
2. Para comportamento novo ou correcao, escreva teste que demonstra o resultado esperado quando o projeto suportar testes automatizados.
3. Cubra logica com maior risco: validacao, transformacao de dados, estados e rotas.
4. Para UI, complemente testes com verificacao visual e navegacao por teclado quando possivel.
5. Execute somente comandos adequados ao projeto e registre resultados reais.

Se o projeto nao tiver testes ou ambiente executavel, diga isso e proponha a verificacao minima manual sem fingir cobertura.

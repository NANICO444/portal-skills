# Documentacao Tecnica

1. Escreva para a pessoa que vai executar ou manter o projeto.
2. Use comandos, caminhos e requisitos confirmados no repositorio.
3. Explique mudancas de contrato, configuracao ou dados quando existirem.
4. Nao registre como concluido algo que nao foi verificado.

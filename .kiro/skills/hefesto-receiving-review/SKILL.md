# Recebendo Review

- Leia o achado no contexto do codigo e dos testes.
- Confirme que o problema existe antes de alterar.
- Se a sugestao resolver um sintoma mas introduzir risco, explique e proponha alternativa.
- Aplique correcoes comprovadas com verificacao correspondente.
- Registre quando o feedback nao procede ou exige decisao do usuario.

## QUANDO USAR
- Ao avaliar se uma ideia de negócio tem potencial real de mercado
- Ao testar hipóteses antes de investir tempo e dinheiro
- Ao estruturar um MVP (Produto Mínimo Viável)
- Ao identificar o cliente ideal e o problema que realmente resolve

## FRAMEWORKS DE VALIDAÇÃO

### 1. Lean Canvas
**Definição:** Versão enxuta do Business Model Canvas para startups e ideias early-stage. Foco em problema, solução e validação de hipóteses.

**Template:**
```
LEAN CANVAS — [Nome da Ideia/Startup] — [Data]

┌─────────────────────┬─────────────────────┬─────────────────────┐
│    PROBLEMA         │   SOLUÇÃO           │  PROPOSTA DE VALOR  │
│                     │                     │  ÚNICA              │
│ Top 3 problemas:    │ Top 3 features:     │                     │
│ 1.                  │ 1.                  │ Mensagem única,     │
│ 2.                  │ 2.                  │ clara e convincente │
│ 3.                  │ 3.                  │ que diferencia      │
│                     │                     │                     │
│ Alternativas        │ Conceito MVP:       │ High-level Concept: │
│ existentes:         │                     │ "X para Y"          │
├─────────────────────┤                     ├─────────────────────┤
│ SEGMENTO DE         │                     │  VANTAGEM           │
│ CLIENTES            │                     │  INJUSTA            │
│                     │                     │                     │
│ Clientes-alvo:      │                     │ O que não pode ser  │
│                     │                     │ copiado ou comprado │
│ Early adopters:     │                     │ facilmente          │
├─────────────────────┼─────────────────────┼─────────────────────┤
│ CANAIS              │    MÉTRICAS-CHAVE   │   ESTRUTURA DE      │
│                     │                     │   RECEITA           │
│ Como atingir        │ O que medir:        │ Modelo de receita:  │
│ clientes:           │ - Aquisição:        │ Preço:              │
│                     │ - Ativação:         │ LTV estimado:       │

│ ESTRUTURA DE CUSTOS          │        FONTES DE RECEITA          │

│ Custos variáveis:            │ Break-even:                       │
│ CAC estimado:                │ Mês 12 MRR/ARR target:            │
└──────────────────────────────┴───────────────────────────────────┘
```

### 2. Business Model Canvas (BMC)
**Definição:** Ferramenta estratégica para mapear o modelo de negócio completo em 9 blocos.

**Template:**
```
BUSINESS MODEL CANVAS — [Empresa/Ideia]

PARCEIROS-CHAVE          ATIVIDADES-CHAVE       PROPOSTA DE VALOR
(quem ajuda?)            (o que fazemos?)       (por que nos escolhem?)
- Fornecedores           - Produção             - Valor criado:
- Parceiros estratégicos - Marketing            - Problema resolvido:
- Co-criadores           - P&D                  - Necessidade atendida:
                         - Atendimento

                         RECURSOS-CHAVE         RELACIONAMENTO COM
                         (o que precisamos?)    CLIENTES
                         - Físicos              - Self-service
                         - Humanos              - Automático
                         - Financeiros          - Comunidade
                         - Intelectuais         - Co-criação

CANAIS                                          SEGMENTOS DE CLIENTES
(como entregamos?)                              (para quem?)
- Direto (site, loja)                           - Mercado de massa
- Indireto (distribuidores)                     - Nicho específico
- Digital vs. Físico                            - Segmentado

ESTRUTURA DE CUSTOS                  FONTES DE RECEITA
(o que custará mais?)                (como ganharemos?)
- Custos fixos:                      - Venda direta
- Custos variáveis:                  - Assinatura
- Economias de escala?               - Licenciamento
                                     - Publicidade
                                     - Freemium → Premium
```

### 3. Problem-Solution Fit
**Objetivo:** Confirmar que o problema existe, é real e vale ser resolvido.

**Checklist de validação:**
```
PROBLEM-SOLUTION FIT — [Ideia]

HIPÓTESE DO PROBLEMA:
"[Segmento de clientes] tem dificuldade em [problema específico]
porque [causa raiz], o que resulta em [consequência negativa]."

VALIDAÇÃO QUALITATIVA (entrevistas com 10-20 potenciais clientes):
Perguntas-chave:
1. "Conte-me sobre a última vez que enfrentou [problema]."
2. "Como você resolve hoje? O que não funciona nessa solução?"
3. "Quanto tempo/dinheiro você gasta com isso hoje?"
4. "Se houvesse uma solução perfeita, como seria?"
5. "O quanto isso te incomoda? (1-10)"

CRITÉRIOS DE VALIDAÇÃO:
[ ] >70% dos entrevistados confirmam que o problema existe
[ ] >50% classificam o problema com urgência ≥7/10
[ ] >60% insatisfeitos com soluções atuais
[ ] Pelo menos 3 entrevistados dispostos a pagar por solução
[ ] Você consegue articular o problema em 1 frase sem usar jargão

RESULTADO: [Validado / Não Validado / Precisa Refinar]
PIVÔ NECESSÁRIO (se aplicável):
```

### 4. Product-Market Fit (PMF)
**Objetivo:** Validar que a solução ressoa com o mercado antes de escalar.

**Template de medição:**
```
PRODUCT-MARKET FIT — [Produto/Serviço]

MÉTODO SEAN ELLIS (Survey PMF):
Pergunta: "Como você se sentiria se não pudesse mais usar [produto]?"
- Muito desapontado: ___% (meta: >40%)
- Um pouco desapontado: ___%
- Não me importaria: ___%
- Já parei de usar: ___%

INDICADORES QUANTITATIVOS:
- NPS (Net Promoter Score): ___ (meta: >50)
- Retenção D30: ___% (meta: >30%)
- Retenção M3: ___% (meta: >20%)
- Taxa de recomendação orgânica: ___%
- Taxa de churn mensal: ___% (meta: <5%)

INDICADORES QUALITATIVOS:
[ ] Clientes descrevem o produto espontaneamente para amigos
[ ] Feedbacks negativos são sobre "quero mais" não "não resolve"
[ ] Clientes ficam bravos quando product fica fora do ar
[ ] Surgem casos de uso que você não havia previsto

STATUS PMF: [Antes do PMF / Buscando PMF / PMF Alcançado]
PRÓXIMOS PASSOS:
```

### 5. TAM / SAM / SOM (Dimensionamento de Mercado)
**Definição:** Estima o tamanho real do mercado endereçável.

**Template:**
```
DIMENSIONAMENTO DE MERCADO — [Ideia/Setor]

TAM — TOTAL ADDRESSABLE MARKET
(Mercado total se você conquistasse 100% do mundo)
Abordagem Top-Down:
- Fonte: [relatório setorial, IBGE, etc.]
- Tamanho: R$ ___ bilhões / ___ milhões de clientes potenciais
Abordagem Bottom-Up:
- Nº de clientes possíveis: ___
- Ticket médio anual: R$ ___
- TAM = ___ × R$ ___ = R$ ___

SAM — SERVICEABLE ADDRESSABLE MARKET
(Mercado que você consegue realisticamente atingir com seu modelo)
- Restrições geográficas:
- Restrições de segmento:
- SAM = ___% do TAM = R$ ___

SOM — SERVICEABLE OBTAINABLE MARKET
(Mercado que você realistically capturará em 3-5 anos)
- Quota de mercado realista:
- Concorrentes estabelecidos:
- SOM = ___% do SAM = R$ ___

VALIDAÇÃO:
- SOM é ambicioso porém atingível?
- O mercado é grande suficiente para justificar o negócio?
```

### 6. Experimentos de Validação (Metodologia Lean)
**Framework de hipóteses → experimentos → aprendizados:**

```
EXPERIMENTO DE VALIDAÇÃO

HIPÓTESE: "Acreditamos que [segmento] vai [comportamento] porque [razão]."

EXPERIMENTO:
- Tipo: [Landing Page / Entrevista / Wizard of Oz / Smoke Test / Concierge]
- O que faremos:
- Duração:
- Custo estimado:

MÉTRICA DE SUCESSO (critério falsificável):
"Se [X% dos usuários fizerem Y ação], a hipótese está validada."

RESULTADO:
- Métrica obtida:
- Hipótese: [VALIDADA / INVALIDADA / INCONCLUSIVA]
- Aprendizado:
- Próxima iteração:

TIPOS DE EXPERIMENTOS:
1. Landing Page: página simples com CTA, mede interesse real
2. Smoke Test: vender antes de construir (pré-venda)
3. Wizard of Oz: simular o produto manualmente
4. Concierge MVP: atender clientes manualmente para aprender
5. Protótipo Clickável: testar UX/fluxo sem código
6. Entrevistas de Problem Discovery: entender dores (qualitativo)
7. Entrevistas de Solution Validation: mostrar protótipo (qualitativo)
8. A/B Test: testar variações de mensagem/oferta
```

### 7. Jobs-To-Be-Done (JTBD)
**Definição:** Entende o "trabalho" que o cliente contrata o produto para fazer.

**Template:**
```
JTBD — [Produto/Ideia]

DECLARAÇÃO DO JOB:
"Quando [situação], eu quero [motivação/objetivo],
para que eu possa [resultado esperado]."

EXEMPLOS:
- Funcional: "Quero organizar minhas finanças para não ficar no vermelho."
- Emocional: "Quero sentir que tenho controle da minha vida financeira."
- Social: "Quero ser visto como alguém responsável por minha família."

MAPA DE FORÇAS (Four Forces):
PULL (atrai para nova solução):
+

PUSH (empurra para longe da solução atual):
+

ANSIEDADES (freiam a mudança):
-

HÁBITOS (prendem na solução atual):
-

IMPLICAÇÕES PARA O PRODUTO:
```

### 8. Validação de Modelo de Negócio (Unit Economics)
**Objetivo:** Confirmar que a unidade econômica é saudável antes de escalar.

**Template:**
```
UNIT ECONOMICS — [Negócio]

CAC — Custo de Aquisição de Cliente:
CAC = Custos totais de Marketing & Vendas / Nº de novos clientes
CAC = R$ ___

LTV — Lifetime Value do Cliente:
LTV = Ticket Médio × Frequência de Compra × Tempo de Retenção
   OU LTV = ARPU / Churn Rate
LTV = R$ ___

RATIO LTV:CAC:
- <1: Negócio insustentável (perde dinheiro por cliente)
- 1-3: Marginal (sustentável mas fraco)
- >3: Saudável (meta para SaaS)
- >5: Excelente
Ratio: LTV/CAC = ___

PAYBACK PERIOD:
Tempo para recuperar CAC = CAC / (Receita Mensal - Custo Variável Mensal)
Payback = ___ meses (meta: <12 meses para SaaS, <6 para e-commerce)

MARGEM DE CONTRIBUIÇÃO:
MC = Preço - Custos Variáveis por Unidade
MC = R$ ___ (___%)

BREAKEVEN:
Breakeven = Custos Fixos / Margem de Contribuição
Breakeven = ___ unidades/mês
```

## CHECKLIST DE VALIDAÇÃO COMPLETA

**Fase 1 — Validação de Problema (antes de construir qualquer coisa):**
- [ ] Lean Canvas preenchido com hipóteses claras
- [ ] 10+ entrevistas de Problem Discovery realizadas
- [ ] >70% dos entrevistados confirmam o problema
- [ ] Alternativas atuais mapeadas e suas limitações entendidas

**Fase 2 — Validação de Solução:**
- [ ] Protótipo/Mockup criado e testado com 5-10 usuários
- [ ] BMC atualizado com aprendizados
- [ ] TAM/SAM/SOM calculados com fontes citadas
- [ ] Unit economics modelados (mesmo que hipotéticos)

**Fase 3 — Validação de Mercado:**
- [ ] MVP lançado com primeiros clientes reais
- [ ] Sean Ellis Survey aplicada (meta: >40% "muito desapontado")
- [ ] Cohort retention analisada
- [ ] LTV:CAC calculado com dados reais

**Fase 4 — Validação de Negócio:**
- [ ] MRR/ARR crescendo consistentemente (>10% MoM early stage)
- [ ] Churn sob controle (<5% mensal para SaaS)
- [ ] Canal de aquisição escalável identificado
- [ ] Modelo de negócio sustentável confirmado
